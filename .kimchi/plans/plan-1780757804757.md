  Following the design discussion, here's the finalized implementation plan:

---

## Goal
Create two beginner-friendly setup guides (one for the developer, one for the friend on Windows) and implement a hybrid signal-format customizer with built-in presets and a custom regex builder.

## Constraints
- No new external Python dependencies beyond what's in `requirements.txt`.
- Existing signal parsing must remain backward-compatible.
- GUI stays Tkinter (no new UI framework).

## Chunks

### Chunk 1 — Setup Guides
**Scope:** Write `SETUP-DEVELOPER.md` and `SETUP-FRIEND.md` in the project root.
**Depends On:** None.
**Accept When:**
- Both files exist and are validated by `python scripts/check_setup_docs.py` (or manual read-through).
- `SETUP-DEVELOPER.md` covers: prerequisites, clone, venv, pip install, run from source, build `.exe`, troubleshooting.
- `SETUP-FRIEND.md` covers: download `.exe`, double-click, setup wizard walkthrough, connect, start bot, troubleshooting.

### Chunk 2 — Format Manager Core
**Scope:** New module `format_manager.py` with `SignalFormatProfile`, `FormatManager`, and regex builder logic.
**Depends On:** None.
**Accept When:**
- `python -m unittest tests/test_format_manager.py -v` passes with ≥10 tests.
- All 4 built-in presets (multi-line, compact, simple, TradingView) parse correctly.
- Custom profile can be created from tagged sample text and matches identical messages.
- `FormatManager.match(text)` returns `Signal` on first successful profile match.

### Chunk 3 — GUI Format Editor Dialog
**Scope:** New `format_editor.py` with `FormatEditorDialog` (tk.Toplevel) for adding/editing custom formats; integrate into `trader_app.py` as a new "Signal Formats" tab.
**Depends On:** Chunk 2.
**Accept When:**
- Dialog opens from the new tab without errors.
- Pasting a sample, tagging tokens, and clicking "Test" shows correct parsed fields.
- Saving a valid custom profile writes to `config.json` and becomes available in the preset list.
- Invalid regex or missing required fields blocks save with a clear error message.

### Chunk 4 — Integration & Fallback
**Scope:** Wire `telegram_bot.py` to use `FormatManager` instead of direct `signal_parser.py` calls; add fallback logging when no format matches; update `README.md` to reference the new setup docs.
**Depends On:** Chunks 2 and 3.
**Accept When:**
- `python trader_app.py` launches and the Signal Formats tab is usable.
- Telegram signals are parsed through `FormatManager`.
- When no profile matches, the log shows: *"Signal received but no format matched. Go to Signal Formats tab to add this channel's format."*
- `README.md` links to `SETUP-DEVELOPER.md` and `SETUP-FRIEND.md`.

## Verification Strategy
- Each chunk runs its own unit tests.
- After Chunk 4, run full test suite: `python -m unittest discover tests -v`.
- Manual smoke test: launch app, open Signal Formats tab, add a custom format, verify parsing.

## Decision Log
| Decision | Rationale |
|---|---|
| Hybrid preset + custom builder | Friendlier than raw regex alone, more flexible than presets only. |
| Tkinter dialog for custom format editor | Consistent with existing GUI; no new dependencies. |
| Store custom profiles in `config.json` | Survives app restarts and updates. |
| Regex builder auto-generates from tags | Users never type regex manually. |

## Risks
| Risk | Likelihood | Mitigation |
|---|---|---|
| Auto-generated regex fails on real-world signal variations | Medium | Test area in dialog lets user validate 3 samples before saving. |
| Friend can't run `.exe` due to Windows SmartScreen | Medium | Documented in `SETUP-FRIEND.md` with steps to click "More info → Run anyway." |
| Backward compatibility break | Low | Default active format is `"builtin-multi"`; hardcoded parser kept as internal preset. |
